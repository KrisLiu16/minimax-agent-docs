## WebSocket 连接断开

**现象**: 语音通话或实时对话中突然断开

**排查步骤**:
1. 检查客户端网络是否稳定
2. 确认 WebSocket 端口未被防火墙拦截
3. 查看服务端日志是否有超时记录
4. 检查负载均衡器的 idle timeout 配置（建议 >= 300s）

## API 请求超时

**现象**: 调用 Agent API 返回 504 或请求超时

**常见原因**:
- **知识库检索慢** — ES 集群负载过高，检查集群状态
- **LLM 响应慢** — 大模型推理延迟，检查 GPU 资源
- **网络链路长** — 跨区域调用延迟高，建议就近部署

**建议配置**:
- 客户端超时：30s
- 网关超时：60s
- 服务端超时：90s

## CORS 跨域问题

**现象**: 前端调用 API 报 CORS 错误

**解决**: 后端已在所有 API 中设置了 CORS 头。如果仍有问题，检查：
1. 请求是否经过了额外的代理层（Nginx 等），代理层可能覆盖了 CORS 头
2. 是否使用了自定义请求头，需要在 `Access-Control-Allow-Headers` 中添加
