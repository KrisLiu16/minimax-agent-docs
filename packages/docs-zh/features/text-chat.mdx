## 功能概述

文本对话是 MiniMax Agent 的核心功能，支持多轮上下文理解和知识库检索增强回答。

## 对话流程

1. 用户发送文本消息
2. Agent 从知识库中检索相关文档
3. 结合上下文历史和检索结果生成回复
4. 回复中标注引用的知识库文档来源

## 支持的消息类型

- **纯文本** — 标准文字消息
- **富文本** — 支持 Markdown 格式的回复
- **卡片消息** — 结构化信息展示（如订单详情、产品推荐）

## 上下文管理

Agent 默认保留最近 20 轮对话历史作为上下文。可以在客服配置中调整：

- **max_turns**: 最大保留轮次（建议 10-30）
- **context_window**: 上下文窗口大小

## 知识库检索

每次用户提问时，Agent 会：

1. 对用户问题进行语义理解
2. 在 Elasticsearch 中检索最相关的 Top-K 篇文档
3. 将检索结果作为参考依据生成回答
4. 在回复中注明引用来源
