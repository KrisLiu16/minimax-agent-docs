---
title: "MaxClaw 技术排障"
---

> ⚠️ **重要：** 遇到任何技术问题，请先提供 chat_id，客服才能进一步排查。
> chat_id 获取：打开对话页面，复制 URL 末尾数字，例如 `https://agent.minimaxi.com/chat?id=370943808623550` 中的 `370943808623550`。

## 一直显示 loading / 不回消息

1. 点击右上角**重启**按钮，再次发送消息
2. 重启失败 → 点击**自动修复**按钮，再次发送消息
3. 仍无效 → 提供 chat_id 联系支持

## 发消息后长时间无回复 / 超时

1. 发送 `/new` 开启新对话，通常立即恢复
2. 若反复出现 → 点击重启

## 返回 400 / HTTP 500 / Internal server error

**最常见原因：积分耗尽。**

1. 检查右上角积分余额是否充足
2. 积分充足但仍报错 → 点击重启
3. 以上均无效 → 提供 chat_id 进一步排查

## 413 / 请求体过大 / Maximum request body size exceeded

对话历史积累过长（约超 1MB）触发，图片识别后尤其常见。

**解决：** 发送 `/new` 清除历史上下文，然后重试。

## Moderation blocked / 对话框无法输入

触发了内容安全审核。

**解决：** 点击右上角齿轮 → **删除后重新创建**。

> 如认为属于误伤，可发送邮件至 MiniMaxAgent@minimax.io，附上 chat_id 申请审核，误伤将退还积分。

## 任务停止 / Task stopped, try a new task

触发了安全审核，任务无法继续。

**解决：** 删除当前 MaxClaw 并重新创建。

## 突然终止 / 对话卡死

点击删除当前 MaxClaw，然后重新创建。

## 飞书配对失败 / 服务繁忙 / 一直显示重启中

原因：提示词不明确，Claw 自我修改配置文件导致损坏。

1. 尝试右上角齿轮 → **自动修复**
2. 无效 → 删除后重新部署，避免频繁修改配置

## 积分为负 / 余额不足无法使用

1. 打开 MaxClaw 网页端，查看右上角积分余额
2. 积分耗尽 → 充值后继续使用
3. 积分充足但仍报错 → 点击重启
4. 仍无效 → 提供 chat_id，充值相关问题联系世东确认
