## 积分消耗规则

MaxClaw 按**实际消耗的 token** 计费，和自己调用 MiniMax API 是同样的逻辑。

**消耗快的常见原因：**
- 每次操作（安装技能、创建 Agent 集群、工具调用）都需要模型参与
- 对话历史越长，每次请求携带的 token 越多
- 联网搜索、图片生成等工具调用带入完整上下文

**节省积分技巧：**
- 完成一个任务后立即发 `/new`，清除当前对话历史（短期记忆）
- 把 `/new` 当做"任务切换键"，每完成一件事就发一次
- 避免在同一对话里做太多不相关的事

## 积分耗尽后的影响

- 积分为零时，对话、工具调用均无法执行
- 定时任务（Cron）和每日推送也会停止
- 充值恢复积分后，定时任务重新正常运行

## Coding Plan / API Key 接入

MaxClaw 支持接入 MiniMax Coding Plan 或按量付费 API Key，可抵扣主要的文本模型积分消耗：

1. 需要 MiniMax Agent Basic 及以上会员
2. 工具调用（图片/视频生成等）仍扣 Agent 账户积分
3. 仅支持在**网页版**配置：https://agent.minimaxi.com/max-claw
4. Coding Plan 有 5 小时窗口限额，超出后需升级套餐或等额度刷新
5. 切换为 Coding Plan 后无法直接回退，如需切回积分模式需删除后重新部署

> 注意：MiniMax Agent 积分与 MiniMax 开放平台余额**完全不互通**，需分别充值。

## 积分查询

如需查询积分使用明细，提供 chat_id 后发送邮件至 MiniMaxAgent@minimax.io 询问。

chat_id 获取：打开对话页面，复制 URL 末尾数字，例如 `https://agent.minimaxi.com/chat?id=370943808623550` 中的 `370943808623550`。
