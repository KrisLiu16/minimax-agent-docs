## How MaxClaw Conversations Work

Every MaxClaw conversation involves the large model processing your full context (history + tool results) to generate a reply.

## /new Command

`/new` clears **short-term memory** (current chat history), starting the next task from a blank context.

- **Cleared:** Current conversation message history
- **Preserved:** MEMORY.md, installed skills, configurations (long-term persistent content)

**Best practice:** Send `/new` after each task before starting the next — this significantly reduces token consumption.

## Longer Context = More Credits

- More history = more tokens per request
- Image recognition and tool calls add large amounts of context
- Use `/new` regularly to keep costs down

## Supported Input Types

- Plain text messages
- Images (image content is carried into context — watch token cost)
- Files (uploaded files can be read and processed by the Agent)

## Built-in Tool Capabilities

| Tool | Description |
|------|-------------|
| Web search | Get real-time information |
| Image generation | Generate images from text descriptions |
| Video generation | Generate videos from text descriptions |
| File operations | Read, create, and modify files |
| Code execution | Run Python and other code |
| Scheduled tasks | Cron automation |

## File Management

- Files generated by MaxClaw default to the `memory/` directory (internal workspace — not visible in the file list)
- Tell the Agent "move the file to the WorkSpace directory" to make it accessible in the file list
